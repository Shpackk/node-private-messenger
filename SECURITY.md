# Security Policy

Do not report vulnerabilities through public issues. Configure a private security contact before
production launch.

Reports should include affected endpoint/component, reproduction steps, impact, and suggested
mitigation. Never include real user keys, tokens, ciphertext, or personal data.

## Non-Claims

This repository is not yet externally audited. It does not implement client E2EE, endpoint security,
traffic-analysis resistance, anonymous routing, or key transparency. Deployment alone does not make
a messenger secure.
